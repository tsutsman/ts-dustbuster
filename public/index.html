Очищення тимчасових файлів

Інтерфейс запускає скрипт dustbuster з поточними налаштуваннями сервера. Перед стартом переконайтеся, що маєте права на видалення вибраних директорій і що CLI вже налаштований.

Що варто знати перед запуском

  • Запускайте інструмент локально: GUI не захищений автентифікацією.
  • Для опції глибокого очищення на Windows потрібні адміністративні права.
  • Глобальні параметри CLI (директорії, виключення, dry-run) застосовуються через конфігураційні файли сервера.
  • Під час виконання не закривайте сторінку — журнал покаже всі кроки очищення.

Запуск очищення

Кнопка нижче надсилає HTTP POST на /clean. Якщо потрібно протестувати без видалення, додайте прапор --dry-run у конфігурації сервера перед відкриттям GUI.

Журнал виконання

Після завершення ви побачите зведення, попередження про доступ та шлях до кожного видаленого елементу. Скопіюйте дані для аудиту, якщо потрібно.
